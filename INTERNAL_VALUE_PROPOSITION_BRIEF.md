## Executive Summary (Paragraph Format)

AWS Cost Anomaly Detective is an educational reference architecture published in aws-samples that demonstrates how customers can build AI-powered operational tools using AWS services. The project teaches developers how to integrate Amazon Bedrock (Claude Sonnet 4.6) with Cost Explorer, CloudTrail, and other AWS services to create event-driven automation. This sample directly drives AWS service adoption by generating ongoing consumption of Bedrock APIs, Lambda executions, Cost Explorer queries, EventBridge events, and DynamoDB storage with every deployment. While AWS's native AI-powered Cost Investigations feature (powered by Amazon Q) provides excellent managed analysis in the AWS Console, this open-source sample serves a complementary educational purpose by showing customers *how* to build similar tools themselves, demonstrating integration patterns they can apply to other operational use cases like security investigations, performance optimization, or compliance automation. The architecture showcases advanced Bedrock capabilities including structured JSON output for programmatic workflows and AI-generated CloudFormation code for automated remediation—patterns not commonly demonstrated in public samples. For AWS, this project increases platform stickiness by teaching customers to build production workloads on Bedrock, positions AWS as the AI/ML platform for operational intelligence, and creates reusable patterns that accelerate customer adoption of AWS AI services beyond simple chatbot implementations.

## Why This Matters for AWS

This sample project drives measurable business value for AWS through increased service consumption and customer retention. Every deployment generates recurring Bedrock API calls (estimated $10+ monthly per customer), Cost Explorer queries (~700/month), Lambda invocations, and associated data transfer costs. By teaching customers how to build AI-powered operational tools, we create "sticky" production workloads that deepen platform commitment and reduce churn. The educational value accelerates Bedrock adoption by providing a real-world reference beyond chatbots, addressing a common customer request for operational AI examples. AWS Solutions Architects, TAMs, and CSMs can use this in customer workshops, demos, and engagements to showcase what's possible when combining AWS AI services creatively. The project complements rather than competes with AWS native features—customers should use Cost Anomaly Detection first, then learn from this sample to build custom extensions and organization-specific automation that integrates with their existing ITSM tools (Jira, ServiceNow, Slack). This approach positions AWS native features as the recommended starting point while demonstrating that AWS services are flexible enough to handle unique customer requirements through open APIs and serverless patterns.

## Target Audience and Use Cases

For AWS colleagues (SAs, TAMs, CSMs), this sample provides a ready-to-demo reference architecture for customer conversations about Bedrock, a hands-on 60-minute workshop for technical enablement, and a foundation for customer engagements requiring custom FinOps automation. For AWS customers, the sample addresses three primary use cases: (1) learning how to use Bedrock APIs in production through working code rather than documentation, (2) building custom extensions to AWS Cost Anomaly Detection that integrate with existing organizational workflows and approval processes, and (3) understanding patterns that can be applied to other operational intelligence scenarios like AI-powered incident response, capacity planning, or resource optimization. For AWS partners and system integrators, the project serves as a reference implementation for customer projects, an accelerator for building Bedrock-powered solutions, and training material for partner engineers learning AWS AI/ML services. The code demonstrates production-ready patterns including error handling, structured logging, IAM least-privilege permissions, and multi-account organization support through cross-account role assumption.

## Technical Differentiation and Educational Value

The architecture showcases advanced integration patterns that customers frequently ask about but struggle to implement. The project demonstrates how to use Bedrock for structured output (JSON schemas with validation) rather than free-form chat responses, enabling programmatic decision-making and automated workflows. It shows how to enrich AI context by combining data from multiple AWS services (Cost Explorer cost trends, CloudTrail API events for who/when/what, Config for configuration history, CloudWatch for performance metrics) to generate high-confidence root cause analysis. The AI-generated CloudFormation code feature demonstrates an advanced use case where Bedrock doesn't just analyze problems but generates executable remediation infrastructure-as-code—a pattern applicable to many operational scenarios. The event-driven architecture (EventBridge → Lambda → Bedrock → Multi-channel notifications) provides a reusable template customers can adapt for security investigations, compliance checks, or performance analysis. By making this code open source in aws-samples, we enable customers to learn from working implementations, adapt patterns to their specific requirements, and contribute improvements back to the community, creating a virtuous cycle that benefits AWS through increased platform engagement and customer success stories.

## Positioning Relative to AWS Native Features

AWS Cost Anomaly Detection with AI-powered investigations (Amazon Q) is the recommended production solution for all customers—it's a fully managed service with AWS support, integrated Console experience, and powerful analysis capabilities. This reference architecture serves a different purpose: it's a teaching tool that shows customers how to use AWS AI/ML services programmatically to build custom operational intelligence solutions. The positioning is strictly complementary, not competitive. Customers should adopt AWS native features first for their core cost management needs, then use this sample to learn how to extend those capabilities with organization-specific automation, custom integration points with existing ITSM systems, and workflows that reflect their unique approval processes and governance requirements. The sample explicitly recommends AWS native features in its documentation and positions itself as educational content similar to other aws-samples projects—it doesn't replace managed services, it teaches customers how to build on the AWS platform effectively. This distinction is critical: we're not providing an alternative to AWS products, we're showing customers why they should build their custom solutions on AWS rather than elsewhere, thereby increasing platform lock-in and service consumption.

For AWS, success is measured through Bedrock API consumption (target: $10,000+ monthly across all deployments), GitHub engagement (target: 500+ stars, 100+ deployments tracked via CloudFormation analytics), and customer success stories showcasing innovative uses of AWS AI services. ## Objection Handling and Internal Guidance

When colleagues ask "Isn't this competing with AWS Cost Anomaly Detection?" the answer is a clear no—this is an educational sample that drives adoption of AWS services, not a product competing for customer spend. Compare it to other aws-samples projects: they don't compete with AWS managed services, they teach customers how to use the platform effectively. This sample explicitly recommends customers use AWS native features first and positions itself as a learning tool for extending those features with custom automation. When customers ask "Why should we use this instead of the native feature?" the guidance is "You should use both—start with AWS Cost Anomaly Detection for managed analysis, then use this sample to learn how to build custom workflows that integrate with your existing tools like Jira and Slack." The project drives Bedrock consumption, increases Cost Explorer API usage, showcases Lambda best practices, and demonstrates EventBridge patterns—all of which benefit AWS. These are transferable patterns applicable to security, compliance, performance, and other operational domains, making the sample a gateway to broader AWS AI/ML adoption.
